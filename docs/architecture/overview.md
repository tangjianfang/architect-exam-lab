# 系统架构概述

> 考点关联：**4+1 视图模型**、**微服务架构风格**、**架构评估（ATAM）**

---

## 1. 系统简介

「在线工具聚合平台」是一个为用户提供多种实用工具（文本处理、图片转换、数据格式化等）的 SaaS 平台。系统采用**微服务架构**，通过 API Gateway 统一对外提供服务，支持工具的动态注册与发现。

本系统同时作为软考系统架构设计师备考的实战项目，每个架构决策都对应具体的考试知识点。

## 3. 架构风格说明

### 3.1 微服务架构（Microservices Architecture）

> **考点：架构风格** — 微服务 vs SOA vs 单体架构的权衡

系统采用微服务架构，将功能拆分为独立的服务单元：

```
┌─────────────────────────────────────────────────────────────────┐
│                          客户端（Browser/Mobile/CLI）             │
└─────────────────────────────────┬───────────────────────────────┘
                                   │ HTTPS
                                   ▼
┌─────────────────────────────────────────────────────────────────┐
│                       API Gateway（:3000）                        │
│            路由 + 认证验证 + 限流 + 熔断 + 日志                    │
└────────┬───────────────────────┬────────────────────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐  ┌──────────────────────┐  ┌──────────────────┐
│  Auth Service   │  │   Tool Registry      │  │  Tool Services   │
│    (:3001)      │  │     (:3002)          │  │   (:3003~)       │
│  JWT + RBAC     │  │  服务注册与发现        │  │  具体工具逻辑     │
└────────┬────────┘  └──────────┬───────────┘  └────────┬─────────┘
         │                      │                        │
         └──────────────────────┼────────────────────────┘
                                 │
                    ┌────────────┴────────────┐
                    │                         │
                    ▼                         ▼
         ┌────────────────┐        ┌────────────────────┐
         │  PostgreSQL    │        │      Redis         │
         │  用户/工具数据  │        │  Session/缓存       │
         └────────────────┘        └────────────────────┘
```

### 3.2 架构风格组合

| 架构风格 | 应用范围 | 考点 |
|---------|---------|------|
| **微服务架构** | 整体系统 | 服务自治、独立部署、去中心化 |
| **API Gateway 模式** | 边缘层 | 横切关注点聚合、统一入口 |
| **事件驱动架构** | 服务间异步通信 | 松耦合、消息队列 |
| **分层架构** | 单个服务内部 | Controller→Service→Repository |
| **CQRS（预留）** | 高读写分离场景 | 命令查询职责分离 |

## 5. 核心质量属性目标

> **考点：质量属性（Quality Attributes）** — ATAM 评估方法

| 质量属性 | 目标指标 | 实现机制 |
|---------|---------|---------|
| **性能** | API P99 响应 < 200ms | 缓存（Redis）、连接池 |
| **可用性** | 99.9%（月停机 < 43min） | 熔断器、健康检查、多实例 |
| **安全性** | OWASP Top 10 全覆盖 | JWT 认证、输入验证、HTTPS |
| **可修改性** | 新增服务不影响现有服务 | 松耦合微服务、API 版本化 |
| **可测试性** | 单元测试覆盖率 ≥ 80% | 依赖注入、Mock、Vitest |
| **可部署性** | 5分钟内完成滚动更新 | Docker、K8s Rolling Update |
