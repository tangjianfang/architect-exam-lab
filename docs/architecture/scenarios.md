## 2. 核心场景详述

### 场景 1：用户登录并调用工具（UC-02 + UC-03）

这是系统的**主干场景**，覆盖 Gateway、Auth Service、Tool Registry、Cache 的完整链路。

**前置条件：**
- 用户已注册账户
- 工具服务已在 Tool Registry 中注册

**正常流程：**
```
用户          Gateway         Auth Service     Tool Registry    Cache(Redis)
 │                │                │                │               │
 │─[POST /login]──→                │                │               │
 │                │─[验证路由，无需 Token]            │               │
 │                │─[POST /auth/login]──→            │               │
 │                │                │─[查询用户]       │               │
 │                │                │←─[用户数据]      │               │
 │                │                │─[bcrypt 验证密码]│               │
 │                │                │─[生成 JWT]      │               │
 │                │←─[{accessToken, refreshToken}]─  │               │
 │←─[200 + JWT]───                 │                │               │
 │                │                │                │               │
 │─[GET /tools/text-converter]──→  │                │               │
 │  Header: Authorization: Bearer <token>           │               │
 │                │─[Auth MW: 验证 JWT]─→            │               │
 │                │                │─[verify token] │               │
 │                │                │←─[用户信息]     │               │
 │                │─[Rate Limiter: 检查令牌桶]        │               │
 │                │─[Cache: 查询工具路由缓存]──────────────────────────→
 │                │                                  │←─[缓存未命中]─
 │                │─[GET /registry/tools/text-converter]──→          │
 │                │                                  │─[查询工具信息]  │
 │                │                                  │←─[工具端点]    │
 │                │─[写入缓存（TTL: 5分钟）]────────────────────────────→
 │                │─[代理到工具服务]                   │               │
 │←─[200 + 结果]──                 │                │               │
```

**架构关联：**
- **逻辑视图**：Controller → Service → Repository 分层
- **进程视图**：异步 I/O，Event Loop 驱动
- **物理视图**：请求跨越多个 Docker 容器
- **开发视图**：各层职责分离，共享 types 包

### 场景 2：限流触发（令牌桶算法）

**触发条件：** 同一 IP 在 1 分钟内请求超过 100 次

```
用户（第101次请求）    Gateway
      │                  │
      │─[GET /api/...]──→│
      │                  │─[Rate Limiter MW]
      │                  │  检查令牌桶：令牌数 = 0
      │                  │  令牌桶已空，拒绝请求
      │←─[429 Too Many Requests]
      │                  │  响应头：
      │                  │  X-RateLimit-Limit: 100
      │                  │  X-RateLimit-Remaining: 0
      │                  │  X-RateLimit-Reset: <timestamp>
```

**架构意义：**
> 限流在 Gateway 层实现，而非业务服务层，体现了 **横切关注点（Cross-cutting Concerns）** 的架构原则。

### 场景 3：服务熔断（Circuit Breaker）

**触发条件：** Auth Service 连续 5 次请求失败

```
         Gateway          Auth Service（故障中）
            │                    │
请求1─→     │─[认证验证]──────────→×  失败 (timeout)
请求2─→     │─[认证验证]──────────→×  失败
请求3─→     │─[认证验证]──────────→×  失败
请求4─→     │─[认证验证]──────────→×  失败
请求5─→     │─[认证验证]──────────→×  失败
            │  失败计数 = 5，超过阈值
            │  熔断器：CLOSED → OPEN
            │
请求6─→     │  熔断器状态：OPEN
            │  直接返回 503（不访问 Auth Service）
            │←[503 Service Unavailable]
            │
... (30秒后) ...
            │  熔断器：OPEN → HALF-OPEN
请求N─→     │─[试探性请求]─────────→✓  成功！
            │  熔断器：HALF-OPEN → CLOSED
            │  服务恢复正常
```

**状态机转换图：**
```
    [初始]
       │
       ▼
  ┌─────────┐   失败次数 ≥ 阈值    ┌──────────┐
  │  CLOSED │─────────────────────→│   OPEN   │
  │（正常）  │                      │（熔断中） │
  └─────────┘                      └──────────┘
       ↑                                │
       │ 探测成功                        │ 等待 30 秒
       │                                ▼
       │                          ┌────────────┐
       └──────────────────────────│ HALF-OPEN  │
         探测失败，回到 OPEN        │（探测中）   │
                                  └────────────┘
```

## 3. 场景与架构视图映射

| 场景 | 逻辑视图 | 进程视图 | 物理视图 | 开发视图 |
|------|---------|---------|---------|---------|
| 用户登录 | User/Auth 领域模型 | Event Loop 异步处理 | gateway+auth 容器 | auth.controller/service/repo |
| 工具调用 | Tool 领域模型 | Redis 缓存 I/O | 多容器协作 | 缓存策略模式 |
| 限流 | 无（横切关注点）| 令牌桶计算 | gateway 容器内 | rate-limiter.ts |
| 熔断 | 无（横切关注点）| 状态机转换 | gateway 容器内 | circuit-breaker.ts |
